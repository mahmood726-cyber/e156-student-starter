## E156 Format (7-sentence contract)
- Exactly 7 sentences (S1-S7), at most 156 words, single paragraph
- No citations/links/metadata in body; one named primary estimand
- S1=Question ~22w, S2=Dataset ~20w, S3=Method ~20w, S4=Result ~30w, S5=Robustness ~22w, S6=Interpretation ~22w, S7=Boundary ~20w
- Spec: `C:\E156\spec.md` and `C:\E156\docs\E156_v0.2_SPEC.md`

## Workbook Protection (NON-NEGOTIABLE)
File: `C:\E156\rewrite-workbook.txt`
1. **CURRENT BODY** = AI version. Update freely - UNLESS `SUBMITTED: [x]` (frozen).
2. **YOUR REWRITE** = Mahmood's rewrite. **NEVER touch** - even if it violates rules.
3. **SUBMITTED** = `[ ]` or `[x]`. Only Mahmood toggles this.
4. New entries: include CURRENT BODY + empty YOUR REWRITE + `SUBMITTED: [ ]` + update total count.

## Pipeline: Project -> E156 -> GitHub -> Pages
1. Build tool/app with tests
2. Write CURRENT BODY, leave YOUR REWRITE blank
3. Push to GitHub with README + `E156-PROTOCOL.md` (project name, dates, body, dashboard link)
4. Enable GitHub Pages for HTML dashboards
5. Update `C:\ProjectIndex\INDEX.md` + `rewrite-workbook.txt`
